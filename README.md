# empathy-onboarding

Скилл для Claude Code / Cursor. Онбордит сотрудника компании:

0. **AI Mandate** — 4 обязательных правила (выведены из находок AI-аудита команды: кросс-проверка финансовых цифр, кросс-проверка моделей, AI не источник истины, подтверждение что вложение реально обработано) зашиты в скилл жёстко, попадают в `COMPANY.md` каждого без исключений
1. **Контекст роли** — короткое интервью (роль, метрика, задачи) → `COMPANY.md`
2. **Архитектура ассистента** — синтез из ТРЁХ источников: индивидуальный отчёт M0-аудита (поведенческий факт), отчёт Talent Q (общий профиль компетенций), JSON-профиль из `empathy-behavior-profile-tool.html` (форс-чойс тест по нашим 15 моделям поведения) → **инструментальные** компенсирующие поведения агента (привязаны к конкретным скиллам/правилам, не к прилагательным) с пометкой источника → `AGENT-PERSONALITY.md`

## Установка

В Claude Code / Cursor в рабочей папке:

```
install skill from https://github.com/sergevalovoi/empathy-onboarding
```

## Требования

- Индивидуальный отчёт M0-аудита (Empathy-AI-Audit.html, раздел «Зона роста») — опционально
- Отчёт Talent Q в любом формате (PDF/текст/скриншот) — опционально
- JSON-профиль из [empathy-behavior-profile.pages.dev](https://empathy-behavior-profile.pages.dev) (форс-чойс тест, 15 моделей поведения, ~16 минут, 50 раундов) — опционально, но самый прямой и инструментальный сигнал из трёх. С 09.07.2026 инструмент сам считает и профиль ассистента (presence/зеркало vs compensate/компенсация) рядом с профилем человека — экспортируй файл `*-with-agent.json`, он уже несёт обе части
- Работает и без всех трёх — тогда Фаза A выдаёт `COMPANY.md` только с 4 правилами AI Mandate + ролью, без персонализации `AGENT-PERSONALITY.md`

## Приватность

Результаты Talent Q и файлы `COMPANY.md` / `AGENT-PERSONALITY.md` остаются локально у пользователя. Скилл (эта логика/промпты) — публичный, персональные данные сотрудников в репозиторий никогда не попадают.
